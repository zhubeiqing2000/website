/ 方向3：工程化
论文现状
  • 数据：2013-2019年NOAA POES+MetOp-A历史数据（静态文件）
  • 处理：Python批量脚本，离线处理
  • 推理：模型训练后手动评估，无在线推理
深化方案：四层实时数据Pipeline架构
[ 数据源层 ]
NOAA SWPC API (实时太阳风) OMNI数据集 (地磁指数) 张衡一号/澳门科学一号 → 数据采集服务 (定时拉取/流式订阅)
[ 数据处理层 ]
原始数据 → 时间对齐 (60min窗口) → 缺失值插值 → 去重
→ 时滞特征工程 (120min-12h滚动窗口) → MinMaxScaler 归一化 → 特征向量 (293维)
[ 模型推理层 ]
特征向量 → ANN 模型在线推理 → 4通道×2角度通量预测
模型版本管理 (A/B测试) 置信度估计
[ 应用服务层 ]
预测结果 → 风险等级判定 → 预警规则引擎
WebSocket 实时推送 历史数据存储 可视化 Dashboard API 开放接口
关键技术决策
数据采集：REST API轮询（5分钟间隔）+ 异常重试机制
特征工程：复用论文293维特征向量设计 + 在线滑窗计算
模型服务：Flask + TorchServe，支持批量推理和热更新
存储：InfluxDB（时序数据）+ PostgreSQL（预警记录）
前端：React Dashboard，实时通量曲线 + 风险等级地图
Demo 可验证要素
要素具体要求
真实输入NOAA实时太阳风数据（非模拟）
真实输出4×2通道电子通量预测值 + 风险等级
真实流程数据采集→推理→预警→推送 全链路自动化
可验证结果历史磁暴事件回放 + 实时预警触发
对标岗位能力
岗位要求本方向的体现
Demo能不能跑有真实输入输出、真实业务流程和可验证结果
0-1产品方案设计从离线研究到在线服务的完整工程方案
AI工具使用API集成、模型服务化、自动化流程
简历话术
设计实时空间天气预警数据Pipeline：数据采集服务定时拉取NOAA SWPC实时太阳风API，流式处理模块完成时间对齐、缺失值插值、293维时滞特征工程（复用论文设计），ANN模型通过TorchServe在线推理输出4×2通道通量预测，风险判定引擎基于阈值规则输出四级预警，通过WebSocket实时推送至运营商Dashboard。端到端延迟<5秒，支持历史磁暴事件回放验证和实时预警触发，形成可演示的完整Demo。
